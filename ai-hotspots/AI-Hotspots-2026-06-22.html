🔬 Fugu：Sakana 的混合模型新范式

Aaron Levie (@levie) — 模型路由的未来已来

Levie 高度评价了 Sakana 刚刚发布的 Fugu 模型，认为它是 AI 架构演进的重要方向：

“又一个推动 AI 架构边界的新想法。Sakana 发布了一个模型，它有效地使用模型混合来完成任务。你得到的是一个统一的 API，但工作会被分配给最能胜任该任务的模型。

Fugu 自动处理模型选择、委派、验证和合成。当问题足够简单时直接解决，当问题需要更复杂处理时就协调一个专家模型团队。多代理系统的复杂性永远不会触及你的代码。

这基本上就是当前应用层 AI 产品构建其代理框架的方式，但将其变成一个任何开发者都能直接调用的 LLM，这是一个非常棒的想法。随着前沿模型和开源模型持续创新，能够做好路由的中间层将产生巨大价值。”

🤖 Agent 使用软件的 100 倍未来

Aaron Levie (@levie) — Headless 时代的软件重构

Levie 对 Agent 时代软件需求提出了系统性的思考：

“Agent 使用软件的数量将是人类的 100 倍。

当这种情况发生时，对 Agent 行为的护栏将变得极其重要——防止数据泄露或误改信息、需要权威的数据源、日志和审计追踪、让人与 Agent 通过系统协作的能力，以及更多。

任何一个 Agent 任务可能拉取的数据，都比用户一个月操作的还多。因此，当软件走向 headless 时，许多软件类别的使用量和价值都会大幅提升。

Agent 最终会用我们的 CRM 数据、企业文档和知识库、分析数据和其他信息，远超人类过去的使用量。

能够向这种 headless 交互模式演进，并且拥有支持这一转变的商业模式和技术策略的平台，将占据最佳位置。”
