📌 今日热点

2026年5月28日，AI 开源生态持续火热。OpenBMB 发布任务型 AI Agent 平台 PilotDeck，OpenSquilla 以”同预算更高智能密度”理念引发关注；ElevenLabs 推出 Music v2 模型可实现曲风即时切换；Amazon 将 Alexa for Shopping 技术开放给第三方零售商。GitHub 上多款 AI/LLM 学习资源与工具项目收获大量星标，社区对 Agent 技术的热情不减。

🔥 GitHub 热门 AI 项目

1. yao-open-prompts — 中文 AI 提示词库

  • Stars: ⭐ 2,314 | Forks: 355
  • 语言: Python | 协议: MIT
  • 链接: https://github.com/yaojingang/yao-open-prompts

这是一个系统性收录高质量中文 AI 提示词的开源项目，覆盖工作、学习、内容创作、营销和生活五大场景。每个提示词都经过手工优化，直接复制即可投入使用。对于不擅长 Prompt Engineering 的中文用户来说，它是一个立即可用的生产力工具箱。

适用场景：文案写作、数据分析、编程辅助、学习规划、营销策划等。

2. how-to-train-your-gpt — 从零构建现代 LLM

  • Stars: ⭐ 2,117 | Forks: 270
  • 语言: Jupyter Notebook | 协议: MIT
  • 链接: https://github.com/raiyanyahya/how-to-train-your-gpt

项目口号”每一行都有注释，用五岁小孩都能懂的方式解释”——它不是一个库，而是一整套从零搭建 GPT 的教程，涵盖 Tokenization、Attention 机制、Transformer 架构、预训练、微调等完整链路。使用 PyTorch 实现，代码整洁且注释详尽，是深度学习入门者的绝佳教材。

3. OpenSquilla — 高智能密度的 Token 高效 AI Agent

  • Stars: ⭐ 2,077 | Forks: 139
  • 语言: Python | 协议: Apache 2.0
  • 链接: https://github.com/opensquilla/opensquilla

OpenSquilla 的核心理念很直接：用同样的 Token 预算，产出更高的智能密度。它通过优化 Agent 的上下文管理、记忆系统和 MCP（Model Context Protocol）集成，减少无效 Token 消耗，让模型在有限的上下文窗口内”思考得更深”。项目与 OpenClaw 生态有联动，支持技能（Skills）插拔式扩展。

4. PilotDeck — 面向任务的 AI Agent 生产力平台

  • 开发者: OpenBMB（清华团队）
  • 描述: 任务导向的 AI Agent 生产力平台
  • 链接: https://github.com/OpenBMB/PilotDeck

OpenBMB（Open Lab for Big Model Based）推出 PilotDeck，定位为”任务型 AI Agent 的平台化解决方案”。它将多种大模型能力整合到统一的 Agent 框架中，支持任务的拆解、编排、执行和回溯。与市面上偏向对话式的 Agent 不同，PilotDeck 更强调结构化任务执行——用户定义目标，Agent 自动规划子任务并调用工具完成。

5. get-shit-done-redux — 元提示与规范驱动开发

  • Stars: ⭐ 1,484 | Forks: 85
  • 语言: JavaScript | 协议: MIT
  • 链接: https://github.com/open-gsd/get-shit-done-redux

主题标签包括 claude-code、context-engineering、meta-prompting、spec-driven-development。这是 GSD（Get Shit Done）精神在新 AI 时代的延续——通过精心设计的元提示和上下文工程，让 Claude Code 等 AI 编码工具更好地理解项目全貌，实现规范驱动的自动化开发。它不只是提示词集合，更像是一套AI 协作方法论。

6. awesome-agentic-ai-zh — 中文 Agentic AI 学习地图

  • Stars: ⭐ 1,800+（估计）
  • 语言: Markdown
  • 链接: https://github.com/WenyuChiou/awesome-agentic-ai-zh

一份结构化的三语（繁中/简中/English）Agentic AI 学习路线图，覆盖从 LLM 基础到多 Agent 系统的 8 个阶段，收录 145+ 精选项目和实操练习。对有志于系统学习 AI Agent 的开发者来说，这是一份不可多得的索引资源。

🛠️ 新工具 / 产品速览

ElevenLabs Music v2 — 曲风即时切换

ElevenLabs 发布了第二代 AI 音乐生成模型 Music v2，最大亮点是可以在同一首歌中从歌剧无缝切换到重金属，同时支持更快的说唱节奏和非音乐音效。该模型仅使用授权数据进行训练，可商用。

Amazon 开放 AI 购物助手技术

Amazon 首次将其 Alexa for Shopping 背后的 AI 技术打包为 AWS 市场产品，开放给 Kate Spade 等第三方零售商。该方案可提供个性化推荐（带图片和定价）、回答店内政策问题，相当于为电商平台提供了一套”即插即用”的 AI 导购解决方案。

Figma Make 正式 GA

Figma Make 现在可连接生产代码仓库，在可视化界面中直接编辑和构建真实软件。团队可以在 Figma 中完成从设计到代码的完整闭环，新增的编辑面板支持精确的设计调整（布局、颜色、字号等）。

Vertu Alphafold — 奢侈品 AI 手机

Vertu 推出售价 6,880 美元起的折叠屏手机 Alphafold，内置 AI 助理 Hermes。用上了去年的 Snapdragon 8 Gen 4 芯片、500 万像素长焦镜头（你没看错，50 万像素的十倍），以及鳄鱼皮/黄金/钻石等奢侈品选项。

Resolve AI — 多 Agent 生产故障排查

Resolve AI 推出多 Agent 调查系统，同时派出多个专业 Agent 并行追查不同假设，相互交叉验证后构造完整因果链。相比单 Agent 模式，根因定位准确率提升一倍以上。

💡 技术洞见

趋势一：Agent 基础设施层的 Token 经济学

从 OpenSquilla 的”Token-Efficient”设计到 get-shit-done-redux 的上下文工程，本周多个项目都在指向同一个方向——Token 不是无限的，如何用好每个 Token 才是 Agent 落地的关键。

当前主流 LLM 的上下文窗口虽然已经扩展到 128K-1M Token，但推理成本与序列长度呈非线性增长。OpenSquilla 通过 Context Compressor 压缩历史对话，通过 MCP 协议精准调用外部工具而非让模型自己”猜”，本质上都是在做 Token 预算管理。

趋势二：从”对话”到”任务编排”的 Agent 范式迁移

PilotDeck 和 Resolve AI 都体现了同一个趋势——Agent 不再是简单的对话机器人，而是任务执行引擎。PilotDeck 强调结构化的任务拆解和编排，Resolve AI 使用多 Agent 并行假设验证。

这背后的逻辑是：单一大模型做推理很强，但做”事”（尤其是需要多步骤、多工具、多轮校验的任务）时，需要框架层面的支撑。任务编排 + Agent 协作将成为下一阶段 AI 工程化的核心方向。
